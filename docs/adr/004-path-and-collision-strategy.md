# ADR-004: ローカルパスと名前衝突の戦略

- ステータス: 採用
- 日付: 2026-07-11

## コンテキスト

Notion の階層を Obsidian の Markdown ファイル階層へ写像する必要がある。タイトルは可変で衝突・不正文字・Unicode 正規化の問題を含む。過去ツールでは子ページの平坦化・タイトルを識別子にする・改名/移動時に旧ファイルが残る問題があった。

## 決定

Notion ページ ID を唯一の安定識別子とし、毎回「期待するローカルパス」を計算する。

### 階層写像
- ページ本体は `<title>.md`、その子は `<title>/` フォルダ配下に置く（page + child folder 構成）。

```
Notion: 開発 > Rails > Turbo
Obsidian:
  Notion Mirror/開発.md
  Notion Mirror/開発/Rails.md
  Notion Mirror/開発/Rails/Turbo.md
```

### サニタイズ
- 危険文字を安全化: `/ \ : * ? " < > |`、制御文字、末尾空白、末尾ピリオド、`.` / `..`、Windows 予約名。
- Unicode は **NFC 正規化**（macOS の NFD 問題を吸収）。表示上似た別名も考慮。
- パスセグメント長に安全な上限を設ける。極端に長いタイトルは切り詰めて ID suffix を付与。
- 空タイトルは `Untitled--<id短縮>.md`。

### 衝突回避
- 同一親配下で名前が衝突する場合**のみ**、決定論的に ID の短縮値を付与する（例: `メモ.md` と `メモ--a1b2c3d4.md`）。
- 衝突の有無で同期のたびに名前が揺れないよう、一度採用した衝突回避名を state DB で安定管理する。

### 安全性
- パス結合は必ず正規化し、managed directory の外へ出ないことを検証する（`../` トラバーサル防止）。
- symlink を通じて managed 外へ書き込まない。
- 管理対象外ファイルとパスが衝突した場合、上書きせず衝突を報告し安全な別名を使う。

## 影響

- 改名は同一 ID の既存ファイルを特定し MOVE として扱う（旧ファイルを残さない）。移動も新階層へ MOVE（旧階層を残さない）。詳細は ADR-005 / Phase 6。
